# FridgeAI 云服务器部署指南

> 适用: Linux 云服务器 (Ubuntu 20.04+/Debian 12+, 最低 2C4G)
> 更新: 2026-07-14

## 架构概览

```
Nginx :80/:443 (SSL终止 + 静态文件 + 反代)
  ├── /api/*  → FastAPI :8000
  ├── /ws/*   → FastAPI :8000 (WebSocket)
  ├── /static/ → 食材图片 (Nginx直接提供, 7天缓存)
  └── /       → 前端H5 (Nginx静态文件)

FastAPI Backend (uvicorn 1 worker)
  ├── Neo4j Community (独立容器, 512MB heap)
  ├── Milvus Lite (嵌入式, 同进程)
  └── SQLite (checkpoints.db, 本地文件)
```

**外部依赖**: DeepSeek API (LLM) / OneNET IoT (可选) / Pexels API (可选, 食材图片)

## 1. 环境准备

### 1.1 服务器要求

| 资源 | 最低 | 推荐 |
|------|------|------|
| CPU | 2核 | 4核 |
| 内存 | 2GB | 4GB+ |
| 磁盘 | 20GB | 40GB+ |
| 系统 | Ubuntu 20.04 / Debian 12 | — |

### 1.2 安装 Docker

```bash
# 官方一键安装
curl -fsSL https://get.docker.com | sh

# 将当前用户加入 docker 组 (免 sudo)
sudo usermod -aG docker $USER
# 重新登录使生效

# 验证
docker --version
docker compose version
```

### 1.3 开放端口

云服务器安全组需开放:

| 端口 | 协议 | 说明 |
|------|------|------|
| 22 | TCP | SSH |
| 80 | TCP | HTTP (Nginx) |
| 443 | TCP | HTTPS (Nginx) |

**安全提醒**: Neo4j (7474/7687) 和 FastAPI (8000) 端口绑定在 `127.0.0.1`，不对外暴露。

## 2. 部署步骤

### 2.1 克隆代码

```bash
git clone https://github.com/silver4444-xs/FridgeApp.git
cd FridgeApp
```

### 2.2 配置环境变量

```bash
# 首次运行: 生成配置模板
bash deploy/scripts/deploy.sh
# 脚本检测到 .env.production 不存在, 会自动创建模板并退出
```

编辑 `deploy/.env.production`，填入真实凭证:

```bash
vim deploy/.env.production
```

**必填项**:

| 变量 | 说明 | 示例 |
|------|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥 | `sk-xxxxxxxx` |
| `NEO4J_PASSWORD` | Neo4j 数据库密码 | `YourStrongPassword123` |

**可选项** (不填则使用默认值或回退):

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEEPSEEK_BASE_URL` | `https://api.deepseek.com/v1` | LLM API 地址 |
| `LLM_MODEL` | `deepseek-v4-flash` | 主 Agent 模型 |
| `ONENET_PRODUCT_ID` | (空) | 不填则 IoT Relay 自动降级 |
| `ONENET_DEVICE_SECRET` | (空) | 同上 |
| `ONENET_ACCESS_KEY` | (空) | 同上 |
| `PEXELS_API_KEY` | (空) | 不填则跳过食材图片下载 |
| `API_KEY` | (空) | 不填则无 API 认证 |
| `LANGSMITH_TRACING` | `false` | 设为 `true` 开启全链路追踪 |

### 2.3 一键部署

```bash
bash deploy/scripts/deploy.sh
```

脚本自动完成:
1. 环境检查 (Docker版本/内存/磁盘)
2. 构建 Backend Docker 镜像 (首次约 5-10 分钟, 包含 sentence-transformers 模型下载)
3. 启动 Neo4j → 等侯健康 → 启动 Backend → 启动 Nginx
4. 健康检查轮询 (最多 2 分钟)
5. 输出部署摘要

### 2.4 验证部署

```bash
# 健康检查 (无需 API Key)
curl http://localhost:8000/api/health-public
# → {"status":"ok","recipes_count":323,"index_size":...}

# 服务状态
docker compose -f deploy/docker-compose.yml ps
# 三个服务应均为 healthy / running

# 查看日志
docker compose -f deploy/docker-compose.yml logs -f backend
```

## 3. 前端部署

### 3.1 构建 H5 (在开发机)

```bash
cd Frontend

# 首次: 初始化 npm 项目
npm init -y
npm install @dcloudio/vite-plugin-uni vite vue@3

# 构建
npx vite build --mode production
# 输出: dist/build/h5/
```

### 3.2 上传到服务器

```bash
# 从开发机上传 (替换 YOUR_SERVER_IP)
scp -r dist/build/h5/* root@YOUR_SERVER_IP:/tmp/fridgeai-h5/

# 在服务器上, 复制到 Docker volume
docker cp /tmp/fridgeai-h5/. fridgeai-nginx:/usr/share/nginx/html/h5/
```

### 3.3 访问

浏览器打开 `http://YOUR_SERVER_IP`，进入「设置」页面:
- 在「完整URL」输入框填入 `http://YOUR_SERVER_IP` 或域名
- 点击「保存域名」

## 4. 配置域名与 HTTPS

### 4.1 DNS 解析

在域名管理后台添加 A 记录:

```
类型: A
主机记录: @ (或子域名如 fridge)
记录值: YOUR_SERVER_IP
TTL: 600
```

### 4.2 配置 Nginx

编辑 `deploy/nginx/conf.d/fridgeai.conf`:

1. 将 `# server_name your-domain.com;` 取消注释并改为实际域名
2. 取消 SSL server block 注释 (整段)

### 4.3 获取 SSL 证书

```bash
# 安装 certbot
sudo apt install certbot -y

# 先停止 Nginx (certbot 需要占用 80 端口)
docker compose -f deploy/docker-compose.yml stop nginx

# 获取证书
sudo certbot certonly --standalone -d your-domain.com

# 复制证书到 letsencrypt volume
docker cp /etc/letsencrypt/live/your-domain.com/fullchain.pem /tmp/
docker cp /tmp/fullchain.pem fridgeai-nginx:/etc/letsencrypt/live/your-domain.com/
# 同样复制 privkey.pem

# 重启 Nginx
docker compose -f deploy/docker-compose.yml up -d nginx
```

### 4.4 证书自动续期 (crontab)

```bash
# 添加每月续期任务
crontab -e
# 添加:
0 3 1 * * certbot renew --quiet && docker compose -f /path/to/FridgeApp/deploy/docker-compose.yml restart nginx
```

## 5. 常用运维命令

```bash
# 进入 deploy 目录
cd /path/to/FridgeApp/deploy

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f backend    # Backend 日志
docker compose logs -f neo4j      # Neo4j 日志
docker compose logs -f nginx      # Nginx 日志

# 重启服务
docker compose restart backend
docker compose restart neo4j
docker compose restart nginx

# 停止所有服务
docker compose down

# 完全清理 (删除数据卷! 谨慎!)
docker compose down -v

# 更新镜像 (代码更新后)
git pull
docker compose build backend
docker compose up -d backend

# 进入 Backend 容器
docker compose exec backend bash

# 下载食材图片 (需要 PEXELS_API_KEY)
docker compose exec backend python scripts/fetch_images.py

# 备份数据库
docker compose exec neo4j neo4j-admin database dump neo4j --to-path=/data/dump/
docker cp fridgeai-neo4j:/data/dump/ ./neo4j-backup-$(date +%Y%m%d)/
```

## 6. 故障排查

### 6.1 健康检查失败

```bash
docker compose logs backend | tail -50
```

常见原因:
- **Neo4j 未就绪**: 等侯 30 秒, Neo4j 首次启动需初始化
- **DEEPSEEK_API_KEY 未设置**: 检查 `deploy/.env.production`
- **内存不足**: `docker stats` 查看内存使用, Neo4j 至少需 256MB

### 6.2 WebSocket 连接失败

检查 Nginx 日志:
```bash
docker compose logs nginx | grep ws
```

确保 `deploy/nginx/conf.d/fridgeai.conf` 中 `/ws/` location 包含:
```nginx
proxy_http_version 1.1;
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

### 6.3 前端图片不显示

食材图片需要单独下载:
```bash
docker compose exec backend python scripts/fetch_images.py
```

需要有效的 `PEXELS_API_KEY`。如果不需要图片功能, 可忽略此问题。

### 6.4 内存不足 (OOM)

```bash
# 查看各容器内存使用
docker stats --no-stream
```

优化方案:
- Neo4j 堆: 编辑 `docker-compose.yml`, 降低 `NEO4J_server_memory_heap_max__size` 到 384m
- 确保 `MILVUS_MODE=lite` (嵌入式, 无需独立容器)
- 关闭 LangSmith: 设置 `LANGSMITH_TRACING=false`

### 6.5 无法连接 DeepSeek API

```bash
# 测试 API 连通性
curl -H "Authorization: Bearer $DEEPSEEK_API_KEY" https://api.deepseek.com/v1/models
```

如果使用代理, 修改 `deploy/.env.production` 中的 `DEEPSEEK_BASE_URL`。

## 7. 性能参考

| 场景 | 2C4G 服务器表现 |
|------|----------------|
| 菜谱搜索 (< 1ms) | 内存倒排索引, 几乎无延迟 |
| AI 对话 (流式) | 首 token ~2s, 总耗时 ~15-30s/轮 |
| RAG 检索 (50条) | ~16s/条 (含 LLM 生成) |
| Neo4j 内存 | ~300MB (稳定态) |
| Backend 内存 | ~800MB (含 Milvus Lite + embedding) |
| Nginx 内存 | ~50MB |

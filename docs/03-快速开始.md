# 快速开始

> 5 步本地搭建 FridgeAI 开发环境

## 环境要求

| 依赖 | 最低版本 | 说明 |
|------|---------|------|
| Python | 3.12 | Conda 环境 `cook-rag-1` |
| Neo4j | 5.x Community | 图数据库 |
| Milvus | 2.4+ | 向量数据库 (支持 Lite 模式) |
| Node.js | 18+ | 前端构建 (可选) |
| Docker | 24+ | 容器化部署 (可选) |

## 方式一：本地开发（5 步）

### 第 1 步：克隆项目

```bash
git clone https://github.com/silver4444-xs/FridgeApp.git
cd FridgeApp
```

### 第 2 步：配置 Conda 环境

```bash
conda create -n cook-rag-1 python=3.12 -y
conda activate cook-rag-1
pip install -r Backend/requirements.txt
```

### 第 3 步：配置环境变量

```bash
cp Backend/.env.example Backend/.env
```

编辑 `Backend/.env`，填入必填项：

```ini
# 必填
DEEPSEEK_API_KEY=sk-your-key-here
NEO4J_PASSWORD=your-neo4j-password

# 可选
ONENET_PRODUCT_ID=          # 物联网功能需要
ONENET_DEVICE_SECRET=
ONENET_ACCESS_KEY=
LANGSMITH_TRACING=false     # LLM 追踪
API_KEY=                    # 生产环境 API 认证
```

### 第 4 步：启动依赖服务

**Neo4j**:
```bash
docker run -d --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/your-password \
  neo4j:5-community
```

**Milvus** (二选一):
```bash
# 方式 A: Milvus Lite (嵌入式, 零配置, 推荐开发)
# 无需额外启动，程序自动使用 milvus_lite.db

# 方式 B: Milvus Standalone (独立服务)
docker run -d --name milvus \
  -p 19530:19530 -p 9091:9091 \
  milvusdb/milvus:latest
```

### 第 5 步：启动后端

```bash
cd Backend
uvicorn api.server:app --host 0.0.0.0 --port 8000 --reload
```

首次启动会自动：
1. 连接 Neo4j，创建全文索引
2. 加载图数据，构建菜谱文档
3. 创建 Milvus 向量索引
4. 初始化 Agent + LangGraph

### 验证

```bash
# 健康检查
curl http://localhost:8000/api/health-public

# 菜谱推荐
curl -X POST http://localhost:8000/api/recipes/recommend \
  -H "Content-Type: application/json" \
  -d '{"ingredients":[{"name":"鸡蛋"},{"name":"西红柿"}]}'

# Swagger 文档
open http://localhost:8000/docs
```

## 方式二：Docker Compose（一键部署）

```bash
cd deploy
cp .env.production .env
# 编辑 .env 填入 API Key
docker compose up -d
```

启动容器：`neo4j` (端口 7474/7687) + `backend` (端口 8000) + `nginx` (可选, 端口 80/443)

---

## 前端启动

```bash
cd Frontend
npm install
npm run dev:h5
```

修改后端地址：打开设置页面，填入后端 IP 和端口。

## 常见问题

| 问题 | 解决方案 |
|------|---------|
| Neo4j 连接失败 | 检查 `NEO4J_URI` 和 `NEO4J_PASSWORD`，确认容器已启动 |
| Milvus 连接超时 | 开发环境使用 Lite 模式 (不设置 `MILVUS_HOST`) |
| DeepSeek API 401 | 检查 `DEEPSEEK_API_KEY` 是否正确 |
| sentence-transformers 下载慢 | 设置 `HF_ENDPOINT=https://hf-mirror.com` |
| 端口 8000 被占用 | 修改 `--port` 参数 |
| Conda 环境不存在 | 改为 `python -m venv venv && source venv/bin/activate` |

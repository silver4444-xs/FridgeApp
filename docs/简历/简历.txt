# **项目经历**

## **FridgeAI —— 基于 GraphRAG 与多 Agent 的智能冰箱对话系统**

**技术栈:** Python, LangChain , LangGraph , FastAPI, Neo4j, Milvus, uni-app, WebSocket, Ragas

**项目描述:** 基于 LangGraph + GraphRAG 构建智能冰箱多 Agent 对话系统，融合 Neo4j 知识图谱与 Milvus 向量检索双引擎，实现食材管理、菜谱推荐、食材替换与烹饪问答的全链路智能化，覆盖 IoT 数据采集到 uni-app 多端交互。

- **多 Agent 协作架构:** 设计 3 子 Agent 协作模式（菜谱专家 / 替换专家 / 烹饪专家），通过独立 system_prompt + 专用 tool 集合缩小工具选择空间，配合 5 层中间件（调用限流 / 摘要压缩 / HITL 人机审批 / 模型重试 / 工具重试）保障 Agent 稳定性和可控性，实现复杂烹饪场景下的任务拆解与精准路由。
- **GraphRAG 双引擎检索:** 构建 Neo4j 烹饪知识图谱（食材-菜谱-营养多跳关系）+ Milvus 向量索引，，设计 3 种智能路由策略（hybrid / graph / combined），底层采用双层检索范式（实体级 + 主题级）与 Round-robin 融合，实现多跳关系推理与语义检索的互补。
- **检索增强与同义词扩展:** 构建 323 道菜谱倒排索引 + 60+ 组食材同义词映射（番茄↔西红柿），三层覆盖（索引构建 / 搜索展开 / 匹配校验）提升模糊搜索命中率，结合 jieba 分词与 BM25 关键词匹配保障稀疏查询场景下的召回质量。
- RAG 评测体系: 构建中文烹饪问答 Ragas 评测数据集（覆盖 11 个领域），实现 Context Precision / Recall / Faithfulness / Relevancy / Correctness 五项指标的自动化评测。

## SILVER RESEARCH BOT —— 论文深度研读助手

**技术栈:** Python, asyncio, FastAPI,  PyMuPDF, Vue 3, D3.js, MathJax, Langfuse

**项目描述:** 面向本地可审计场景的 AI Agent 工作台，围绕 Agent 执行过程的可观测性、安全性与长期记忆展开设计，实现对对话、记忆检索、工具调用及 Prompt 注入的全链路追踪与审计。

- Agent 核心循环：从零实现 ReAct循环引擎，含中轮消息注入、崩溃检查点恢复、上下文五步治理（孤立结果清理→缺失回填→微压缩→预算→裁剪）、重复调用拦截、死循环防护， Langfuse 全链路追踪。
- 三层记忆架构：Active Memory（逐轮检索注入）→ Project Memory（跨 session 共享）→ Long-term Memory（持久归档），结合 LLM 重要性评分、语义冲突检测、Ebbinghaus 遗忘曲线实现智能记忆管理。设计 Dream 两阶段后台整合。

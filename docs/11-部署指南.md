# 部署指南

> FridgeAI 生产环境部署 — Docker + Nginx + SSL

完整的部署指南请参阅：[部署指南.md](./部署指南.md)

该文档覆盖以下内容：

## 已覆盖

- **架构概览**: Nginx(:80/:443) → FastAPI(:8000) + Neo4j + Milvus Lite + SQLite
- **环境准备**: 服务器要求 (2C4G)、Docker 安装、文件上传
- **Docker Compose**: neo4j + backend + nginx 三容器编排
- **环境变量**: DeepSeek / Neo4j / Milvus / OneNET 配置
- **Nginx 配置**: SSL 终止、静态文件缓存 7 天、API/WS 反向代理、限流
- **启动顺序**: 拉取镜像 → 初始化 Neo4j → 构建 RAG 索引 → 启动 Nginx
- **验证清单**: 健康检查、API 测试、WebSocket 测试
- **常见问题**: 端口冲突、内存不足、Neo4j 密码、SSL 证书

## 补充建议

### 监控

```bash
# Docker 健康检查
docker compose ps
docker stats

# 日志聚合
docker compose logs -f backend
docker compose logs -f neo4j
```

### 备份

```bash
# Neo4j 备份
docker exec neo4j neo4j-admin dump --database=neo4j --to=/backup/neo4j.dump

# SQLite 备份
cp checkpoints.db checkpoints.db.bak.$(date +%Y%m%d)

# 卷备份
docker run --rm -v fridgeai_neo4j_data:/data -v $(pwd):/backup alpine tar czf /backup/neo4j_data.tar.gz -C /data .
```

### CI/CD (建议)

```yaml
# .github/workflows/deploy.yml
deploy:
  steps:
    - docker build -t fridgeai-backend:${{ github.sha }} .
    - docker push registry/fridgeai-backend:${{ github.sha }}
    - ssh server "docker compose pull && docker compose up -d"
```

### Kubernetes 迁移方向

- Milvus Lite → Milvus Cluster (分布式)
- SQLite → Postgres + PostgresStore
- InMemoryStore → PostgresStore
- Nginx → Ingress Controller
- 静态文件 → CDN / OSS

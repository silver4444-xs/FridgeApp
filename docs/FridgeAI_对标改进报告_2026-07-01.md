## 二、FridgeAI 差距识别

### 2.1 缺失量化评测指标 (致命)

> 对标: "F1值92%、准确率94%", "幻觉率降低40-50%", "准确率74%→92%"

**现状**: 零量化指标。Agent回答质量、RAG检索精度、推荐准确率全部未测量。
**影响**: 无法证明价值、无法追踪优化效果、简历上无法写量化成果。

### 2.2 缺失自动化评测闭环

> 对标: "Badcase → Auto Prompt优化 → Ground Truth回归", "分钟级效果评估"

**现状**: 测试方案写了但零执行。没有标注数据集、没有回归测试、没有自动优化流水线。
**影响**: 每次改进后不知道效果变好还是变坏。

### 2.3 缺失 RRF 融合排序与重排序

> 对标: "RRF融合去重 → Cohere语义重排序"

**现状**: HybridRetrieval 混合 BM25+向量后简单叠加分数，没有 RRF 归一化、没有 reranker。
**影响**: 多路检索结果排序质量差，前K命中率低。

### 2.4 缺失结构化推理 (Structured CoT)

> 对标: "JSON结构化CoT: 意图识别→合规检查→策略生成"

**现状**: Agent 仅依赖自由文本推理，system_prompt 有流程引导但无结构化约束。
**影响**: 推理路径不可控、中间步骤不可解析、决策过程不可审计。

### 2.5 缺失分层记忆架构

> 对标: "短期缓存+长期向量检索"双层记忆, "动态记忆仲裁机制"

**现状**: InMemoryStore 仅存偏好键值对。没有向量化长期记忆、没有记忆冲突检测、没有自动画像。
**影响**: 只能记"忌口=花生"，不能记"用户上周做了红烧肉，喜欢慢炖类"。

### 2.6 缺失安全审查与兜底

> 对标: "前置安全审查模块拦截Prompt注入", "敏感操作人工确认流程"

**现状**: 仅有 HITL 对 save_user_preferences 审批。没有输入安全审查、没有越权检测、没有内容过滤。
**影响**: 已公开仓库 + 无防护 = 可被注入任意Prompt。

### 2.7 缺失引用溯源 (Citation)

> 对标: "引用溯源机制，确保关键细节与检索来源可追溯"

**现状**: Agent 推荐菜谱不标注来源。
**影响**: "这道菜的做法是AI编的还是真实菜谱？"

### 2.8 缺失结构化工具封装 (MCP)

> 对标: "模块化 MCP Server，按服务类型拆分为查询/推荐/评价模块"

**现状**: 8 个 @tool 平铺在主 Agent 上。新工具接入需要改 Agent 代码。

### 2.9 缺失业务可观测性

> 对标: "Langfuse记录Prompt拼接/工具调用轨迹/记忆命中"

**现状**: LangSmith 已配置但未主动使用。没有业务指标监控。

### 2.10 缺失 Self-RAG 多轮检索

> 对标: "Self-RAG多轮检索策略", "检索后自检: 相关度>阈值? 否则换策略"

**现状**: QueryRouter 简单二选一，一次检索不中就失败。

## 四、改进路线

### 阶段1: 评测基线 (1周) — P0

```
1. 执行 v3 框架化测试方案
   ├── Ragas: 50条 → ContextPrecision/Recall/Faithfulness
   ├── DeepEval: 10条 → ToolCorrectness
   └── 记录所有 baseline

2. 标注数据集
   ├── 100条用户真实问题
   └── 标注: expected_tools + expected_answer + 是否幻觉

3. 评测看板
   └── 每次变更自动跑评测 → 对比 baseline
```

**产出**: 第一个量化指标: Faithfulness=X, ToolAccuracy=Y

### 阶段2: 检索提升 (1周) — P1

```
1. RRF 融合 → 预期 ContextPrecision +10%
2. Re-ranker (Cohere/BGE-Reranker) → Top-20精排
```

### 阶段3: Agent优化 (1周) — P1

```
1. 结构化CoT → 预期 ToolAccuracy +15%
2. 引用溯源 → 回答标注菜谱来源
3. Self-RAG多轮检索 → 检索失败自动换策略
```

### 阶段4: 安全+记忆 (1-2周) — P2

```
1. 前置安全审查 (Prompt注入拦截)
2. 分层记忆 (短期+向量长期+自动画像)
3. 自动化评测闭环 (每周趋势报告)
```

> **结论**: FridgeAI 的核心差距不在功能数量，而在**评测体系缺失**(没有量化指标)和**检索质量未经优化**(没有RRF/Reranker/Self-RAG)。优先建立评测基线，用数据驱动后续优化。
